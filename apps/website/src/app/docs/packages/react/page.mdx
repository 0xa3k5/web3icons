# @web3icons/react

React components for Web3 Icons with TypeScript support and tree-shaking optimization.

## Installation

<InstallTabs pkg="@web3icons/react" />

> Requires React 18+

## Individual Components (Recommended)

Import icons directly for optimal bundle size with tree-shaking. All of the icons in the web3icons library are prefixed with their type names (`Token`, `Network`, `Wallet`, `Exchange`).
Token icons are named as `TokenSYMBOL` (uppercase token symbol), and other types are PascalCase `NetworkEthereum`, `WalletMetamask` `ExchangeCoinbase`, `NetworkBinanceSmartChain` etc.

<IndividualReactIcons />

| Prop      | Type                                | Default        | Description               |
| --------- | ----------------------------------- | -------------- | ------------------------- |
| size      | string \| number                    | 24             | Icon size in pixels       |
| variant   | 'mono' \| 'branded' \| 'background' | 'mono'         | Visual style              |
| color     | string                              | 'currentColor' | Color (mono variant only) |
| className | string                              | -              | CSS class name            |

## Dynamic Components

Use dynamic components when the icon is determined at runtime. There is one dynamic icon per supported icon types. `TokenIcon`, `NetworkIcon`, `WalletIcon`, `ExchangeIcon`.

<DynamicReactIcons />

> Dynamic components are _NOT_ tree-shakable and are
> client-only components. They dynamically import icons based on the given props.

All icons support standard SVG props.

## Performance

### Tree-shaking

Import only what you need for optimal bundle size:

```jsx
// Good - only includes BTC icon (~2KB)
import { TokenBTC } from '@web3icons/react'

// Bad - includes ALL icons (~500KB)
import * as Icons from '@web3icons/react'
```

### Static vs Dynamic

Prefer individual components when possible:

```jsx
// Better performance - static import
<TokenBTC size={24} />

// Use only when icon is determined at runtime
<TokenIcon symbol="btc" size={24} />
```

Dynamic components require additional runtime lookup and include all icons in the bundle.

## TypeScript

Full TypeScript support with type definitions:

```tsx
import { TokenIcon } from '@web3icons/react/dynamic'
import type { TVariant } from '@web3icons/common'

interface CryptoCardProps {
  symbol: string
  size?: number
  variant?: TVariant
}

function CryptoCard({
  symbol,
  size = 24,
  variant = 'branded',
}: CryptoCardProps) {
  return <TokenIcon symbol={symbol} size={size} variant={variant} />
}
```

## Component API Reference

### Individual Icon Components

All static icon components share the same interface:

```tsx
interface IconProps extends React.SVGProps<SVGSVGElement> {
  size?: string | number
  variant?: 'mono' | 'branded' | 'background'
  color?: string
  className?: string
}
```

### TokenIcon (Dynamic)

```tsx
interface TokenIconProps extends IconProps {
  symbol?: string
  address?: string
  network?: string
  fallback?: React.ReactNode | string
}
```

```tsx
import { TokenIcon } from '@web3icons/react/dynamic'

<TokenIcon symbol="btc" variant="branded" />

<TokenIcon
  address="0xa0b86991c6218b36c1d19d4a2e9eb0ce3606eb48"
  network="ethereum"
/>

<TokenIcon symbol="unknown" fallback={<div>?</div>} />
```

## Error Handling

Dynamic components handle missing icons gracefully:

```tsx
<TokenIcon symbol="invalid" />

<TokenIcon symbol="invalid" fallback="?" />

<TokenIcon symbol="custom" fallback="https://example.com/fallback.png" />

function SafeIcon({ symbol }) {
  return (
    <ErrorBoundary fallback={<div className="w-6 h-6 bg-gray-200 rounded" />}>
      <TokenIcon symbol={symbol} variant="branded" />
    </ErrorBoundary>
  )
}
```
